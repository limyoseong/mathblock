# Issue #3: 실수 진단 (어디서 틀렸는지 알려주기)

**Type**: AFK
**Label**: ready-for-agent

## Parent

PRD.md - MathBlock 최소 수직 슬라이스

## What to build

오답일 때 단순히 "틀렸다"가 아니라, 정확히 어느 블록에서 처음 잘못됐는지 하이라이트로 표시하는 실수 진단 기능. 이 앱의 핵심 가치.

블록 트리 DFS 비교 알고리즘:
1. 사용자 블록 트리와 모범 블록 트리를 루트부터 ordered DFS로 동시에 순회
2. 같은 위치의 노드 타입(`type`)과 파라미터(`params`)가 일치하면 PASS, 자식으로 진행
3. 불일치하면 해당 노드를 "첫 분기점"으로 마킹하고 시각적으로 하이라이트 (색상 변경)
4. 사용자 트리가 모범보다 짧으면 "여기서부터 블록이 빠졌습니다" 메시지
5. 사용자 트리가 모범보다 길면 "불필요한 블록이 추가되었습니다" 메시지

복수 풀이 경로 지원: 모범 풀이가 배열로 저장되어 있을 때, DFS 순회하며 노드 타입 일치 수를 세서 가장 유사한 경로를 선택하여 비교.

Issue #2의 채점 결과 화면에 통합한다. 오답 시: 실수 진단 하이라이트 + 모범 풀이 side-by-side + 중간값 표시가 함께 나타남.

## Acceptance criteria

- [ ] 오답 시 처음 잘못된 블록이 시각적으로 하이라이트된다
- [ ] 블록이 부족할 때 "블록이 빠졌습니다" 메시지가 표시된다
- [ ] 불필요한 블록이 있을 때 "불필요한 블록이 추가되었습니다" 메시지가 표시된다
- [ ] 복수 풀이 경로가 있는 문제에서 가장 유사한 경로와 비교한다
- [ ] 정답일 때는 하이라이트가 나타나지 않는다
- [ ] DFS 비교 알고리즘에 대한 테스트가 존재하고 통과한다 (정답 트리, 중간 오류 트리, 짧은 트리, 긴 트리, 복수 경로 케이스)
