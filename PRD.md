# PRD: MathBlock - 편입 수학 블록 풀이 앱 (최소 수직 슬라이스)

## Problem Statement

편입 수학을 공부할 때, 기존 방법(문제집 + 해설 PDF + 유튜브)은 "답"을 보여주지만 "풀이 과정"을 직접 체험하게 하지 않는다. 콴다 같은 앱은 사진을 찍으면 답을 주지만, 왜 그 답이 나오는지 과정을 이해하게 만들지는 않는다.

핵심 문제는 **"어디서 틀렸는지 모른다"**는 것이다. 문제를 풀다 틀리면 "답이 다르다"는 것만 알 수 있고, 구체적으로 어느 단계에서 어떤 실수를 했는지 알 수 없다.

## Solution

수학 풀이 과정을 스크래치(Scratch) 블록 코딩 형식으로 표현하는 웹앱. 사용자는 타이핑 없이 블록을 드래그해서 끼워넣는 것만으로 문제를 풀고, "실행" 버튼을 누르면 정답 비교와 함께 **정확히 어느 단계에서 풀이가 갈라졌는지** 보여준다.

"답이 틀렸다"가 아니라 **"3번째 블록에서 치환을 잘못 적용했다"**를 알려주는 것이 핵심 가치. 수학을 디버깅하는 경험.

MVP는 적분(기초) 5문제로 시작하는 최소 수직 슬라이스. "블록으로 편입수학을 풀 수 있는가?"라는 핵심 가설을 검증하는 데 집중한다.

## User Stories

1. As a 편입 준비생, I want to 적분 문제를 블록 형태로 풀어볼 수 있는 화면을 열고 싶다, so that 문제를 보자마자 바로 풀이를 시작할 수 있다
2. As a 편입 준비생, I want to 문제 목록에서 원하는 문제를 골라서 클릭하고 싶다, so that 내가 연습하고 싶은 문제를 선택할 수 있다
3. As a 편입 준비생, I want to 수식이 LaTeX처럼 깔끔하게 렌더링된 문제를 보고 싶다, so that 교재에서 보는 것과 같은 형태로 문제를 읽을 수 있다
4. As a 편입 준비생, I want to 미리 배치된 블록 일부를 보고 나머지 빈칸을 채우고 싶다, so that 풀이 방향을 가늠하면서 부담 없이 시작할 수 있다
5. As a 편입 준비생, I want to 블록 목록에서 필요한 블록을 찾아 드래그해서 올바른 위치에 넣고 싶다, so that 풀이의 빠진 단계를 내가 직접 채울 수 있다
6. As a 편입 준비생, I want to 블록을 잘못 놓았을 때 다시 빼거나 위치를 바꿀 수 있다, so that 실수를 자유롭게 수정하면서 시도할 수 있다
7. As a 편입 준비생, I want to 모든 블록을 배치한 뒤 "실행" 버튼을 눌러 결과를 확인하고 싶다, so that 내 풀이가 맞는지 즉시 피드백을 받을 수 있다
8. As a 편입 준비생, I want to 내 최종 답이 정답과 일치하는지 바로 보고 싶다, so that 맞았는지 틀렸는지 한눈에 알 수 있다
9. As a 편입 준비생, I want to 틀렸을 때 "어느 블록에서 처음 잘못됐는지" 하이라이트로 표시되는 걸 보고 싶다, so that 내가 어디서 실수했는지 구체적으로 알 수 있다
10. As a 편입 준비생, I want to 채점 후 모범 풀이 블록 배치를 내 풀이 옆에 나란히 보고 싶다, so that 내 접근과 정답 접근을 비교하며 배울 수 있다
11. As a 편입 준비생, I want to 각 블록 단계의 중간 결과값이 표시되는 걸 보고 싶다, so that 풀이의 흐름을 단계별로 따라갈 수 있다
12. As a 편입 준비생, I want to 블록을 하나도 배치하지 않고 실행했을 때 안내 메시지를 받고 싶다, so that 뭘 해야 하는지 알 수 있다
13. As a 편입 준비생, I want to 연결되지 않은 블록이 있을 때 경고를 받고 싶다, so that 실수로 빠뜨린 블록을 알아챌 수 있다
14. As a 편입 준비생, I want to 문제를 다 풀고 나면 다음 문제로 넘어갈 수 있다, so that 연속으로 여러 문제를 연습할 수 있다
15. As a 편입 준비생, I want to 내가 어떤 문제를 풀었고 어떤 문제를 아직 안 풀었는지 볼 수 있다, so that 진도를 파악할 수 있다
16. As a 편입 준비생, I want to 브라우저를 닫았다 다시 열어도 진도가 유지되길 바란다, so that 이어서 공부할 수 있다
17. As a 편입 준비생, I want to 한 문제에 여러 풀이 경로가 있을 때 가장 비슷한 모범 풀이와 비교되기를 원한다, so that 내 접근법에 맞는 피드백을 받을 수 있다
18. As a 편입 준비생, I want to 문제를 틀렸을 때 다시 도전할 수 있다, so that 피드백을 보고 바로 재시도할 수 있다
19. As a 편입 준비생의 친구, I want to 공유받은 URL을 열면 바로 앱을 쓸 수 있다, so that 별도 설치나 회원가입 없이 체험할 수 있다

## Implementation Decisions

### 전체 접근법: 최소 수직 슬라이스 (Approach A)

적분(기초) 5문제만으로 핵심 가설("블록으로 편입수학을 풀 수 있는가?")을 검증한다. 성공하면 기초수학 전체, Anki 연동, 디버깅 모드 등으로 확장.

### 기술 스택

- **Vite + vanilla JS**: React 같은 프레임워크 없이 단순하게. AI 의존 개발에 더 적합
- **Blockly** (`@blockly/blockly`): 블록 드래그앤드롭 에디터. 연결 규칙, 중첩 등 기본 제공
- **KaTeX**: 문제 표시 패널과 결과 패널에서 수식 렌더링. MVP에서 블록 내부는 유니코드 텍스트(예: "∫", "x^n") 사용
- **localStorage**: 사용자 진도 저장
- **Vercel**: 무료 배포. git push시 자동 배포

### 블록 구조

혼합(C) 방식 — 공식 블록 + 연산 블록 + 개별 블록을 계층적으로 분류:

- **공식 블록**: 범용 수학 공식/연산 (`definite_integral`, `apply_power_rule` 등)
- **연산 블록**: 계산 동작 (`multiply`, `add`, `simplify`, `substitute` 등)
- **개별 블록**: 특정 문제의 중간 계산값 (`number_input`, `variable`)
- MVP 커스텀 블록 8종: `definite_integral`, `substitute`, `apply_power_rule`, `simplify`, `add`, `multiply`, `number_input`, `variable`

블록 연결은 수직 스택 + 중첩(블록 안에 블록). 사용자 입력은 타이핑 없이 블록 끼워넣기만.

### 평가 모델: 룩업 기반 (런타임 계산 없음)

모든 콘텐츠가 사전 제작이므로, 런타임 기호 연산(symbolic computation)은 불필요하다.

- 각 문제의 각 블록 조합에 대해 예상 중간값(`expectedOutput`)과 최종 답을 정적 데이터로 미리 정의
- "실행"은 사용자의 블록 배치를 정답 블록 트리와 비교하는 것
- math.js 같은 수식 엔진 불필요. 순수 룩업 + 비교 로직

### 문제 데이터 스키마

각 문제는 트리 구조로 정의된다 (office hours 프로토타입에서 도출):

```json
{
  "id": "integral-001",
  "title": "∫ 2x dx 를 구하시오",
  "subject": "기초수학",
  "katex": "\\int 2x \\, dx",
  "expectedAnswer": "x^2 + C",
  "availableBlocks": ["definite_integral", "apply_power_rule", "multiply", "simplify"],
  "solutionTree": {
    "type": "definite_integral",
    "prefilled": true,
    "children": [
      {
        "type": "multiply",
        "prefilled": false,
        "params": { "coefficient": 2 },
        "children": [
          { "type": "apply_power_rule", "prefilled": false, "params": { "base": "x", "power": 1 }, "expectedOutput": "x^2" }
        ],
        "expectedOutput": "2 * (x^2/2) = x^2"
      }
    ],
    "expectedOutput": "x^2 + C"
  }
}
```

### 실수 진단: 블록 트리 DFS 비교

사용자 블록 트리와 모범 블록 트리를 ordered DFS(깊이 우선 탐색)로 동시에 순회:

1. 루트부터 시작, 같은 위치의 노드 비교
2. 노드 타입과 파라미터가 일치하면 PASS, 자식으로 진행
3. 불일치하면 해당 노드를 "첫 분기점"으로 마킹하고 하이라이트
4. 사용자 트리가 짧으면 "여기서부터 블록이 빠졌습니다" 표시
5. 사용자 트리가 길면 "불필요한 블록이 추가되었습니다" 표시

복수 풀이 경로가 있는 경우, 모범 풀이를 배열로 저장하고 노드 타입 일치 수가 가장 높은 경로를 선택하여 비교.

### 난이도 (MVP 범위)

MVP에서는 1단계만 구현:
- **1단계 (빈칸 채우기)**: 정답 블록 시퀀스가 부분적으로 사전 배치. 사용자가 2~3개의 빠진 블록을 올바른 위치에 드래그하여 채움

향후 확장:
- 2단계: 블록 하이라이트 힌트 (사용할 블록 2~3개 강조)
- 3단계: 완전 자유 풀이 (힌트 없음)

### 에러 상태 처리

- 빈 워크스페이스 제출 → "블록을 배치해 주세요" 메시지
- 연결되지 않은 블록 → 무시하되 경고 표시
- 중복 블록 → 허용 (풀이 경로에 따라 필요할 수 있음)
- 잘못된 블록 순서 → 채점 시 "첫 분기점"으로 표시 (실수 진단과 동일)

### Blockly 워크스페이스 → 내부 트리 변환

Blockly의 `Blockly.serialization.workspaces.save()` 출력을 내부 solutionTree 포맷으로 변환하는 어댑터(`workspaceToTree()`)를 구현. 이 어댑터가 Blockly와 앱 로직을 분리하는 핵심 인터페이스.

### 배포

- Vercel 무료 플랜
- GitHub 연동, git push시 자동 배포
- URL 공유로 친구에게 전달 (설치/회원가입 불필요)

## Testing Decisions

### 테스트 원칙

좋은 테스트란 **내부 코드가 어떻게 짜여져 있는지**가 아니라, **사용자가 보는 결과가 맞는지**를 확인하는 것이다. 예를 들어 "이 함수가 저 함수를 호출했는지"를 테스트하는 게 아니라, "블록을 이렇게 배치하면 정답이라고 나오는지"를 테스트한다.

### 테스트 대상 모듈

1. **블록 트리 비교 (실수 진단)**
   - 이 앱의 핵심 가치. 사용자 트리와 모범 트리를 넣으면 "어디서 틀렸는지" 정확히 찾아내는지 확인
   - 테스트 예시: 정답 트리 입력 → 통과, 3번째 노드가 다른 트리 입력 → 3번째 노드 하이라이트, 블록이 부족한 트리 → "빠졌습니다" 표시, 복수 풀이 중 가장 유사한 경로 선택

2. **워크스페이스 → 트리 변환 (`workspaceToTree`)**
   - Blockly 출력을 내부 포맷으로 바꾸는 어댑터. 이게 틀리면 모든 채점이 틀어짐
   - 테스트 예시: Blockly의 직렬화 데이터를 넣으면 올바른 트리 구조가 나오는지

3. **채점 로직**
   - 최종 답 비교가 정확한지
   - 테스트 예시: 정답 입력 → 통과, 오답 입력 → 실패 + 모범 풀이 데이터 반환

### 테스트하지 않는 것

- Blockly 자체 (외부 라이브러리, 이미 검증됨)
- KaTeX 렌더링 (외부 라이브러리)
- UI 레이아웃, 스타일 (눈으로 확인)
- localStorage 읽기/쓰기 (브라우저 내장 기능)

## Out of Scope

이번 MVP(최소 수직 슬라이스)에서 **의도적으로 빠진 것들**:

- **반복 학습 (Anki/Spaced Repetition)**: 설계 완료됨(DESIGN.md). 핵심 가설 검증 후 추가
- **난이도 2~3단계**: 1단계(빈칸 채우기)로 먼저 검증
- **디버깅 모드 (단계별 실행)**: "다음 단계" 버튼으로 한 블록씩 실행. 확장 단계에서
- **분기(if) 블록**: 조건부 풀이 경로. 복잡도 높음
- **백엔드, 인증, DB**: localStorage만 사용. 나중에 클라우드 전환
- **모바일 반응형**: 웹 데스크톱 먼저
- **콘텐츠 파이프라인 자동화**: 5문제는 수동 제작. 확장 시 AI 파이프라인
- **블록 내 KaTeX 렌더링**: MVP에서는 유니코드 텍스트. v2에서 `FieldKaTeX` 검토
- **기초수학 외 과목**: 미적분, 선형대수, 공학수학은 가설 검증 후
- **리플레이 모드**: 블록 배치 과정 애니메이션 재생 (cross-model에서 제안된 확장 기능)

### 차별점

YouTube에서 스크래치 형식으로 기초수학 푸는 콘텐츠는 있지만, 편입수학(미적분, 선형대수, 공학수학) 수준까지 블록 코딩으로 풀이하는 도구는 없다. 난이도의 갭이 차별점.

### 검증해야 할 전제

1. 블록 조합이 편입수학 난이도에서도 풀이 과정을 충실히 표현할 수 있는가
2. 타이핑 없이 블록 끼워넣기만으로 풀이가 가능한가
3. 5문제 수동 제작이 현실적이고, 향후 확장 시 AI 파이프라인으로 효율화 가능한가

### 열린 질문

1. MVP에서 블록 내부 유니코드 텍스트가 학습에 충분한가, KaTeX 블록 내 렌더링이 필요한가?
2. 5문제의 구체적 범위: 정적분? 부정적분? 치환적분? (콘텐츠 제작 시 결정)
