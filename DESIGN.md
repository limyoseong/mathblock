# MathBlock - 편입 수학 블록 풀이 앱 설계 문서

## 프로젝트 개요

편입 수학 문제를 스크래치(Scratch) 블록 코딩 형식으로 풀 수 있는 웹 앱.
초보 편입 준비생의 부담감을 줄이면서도, 궁극적으로 시험 합격을 위한 실력을 기르는 것이 목표.

## 핵심 컨셉

- 수학 풀이 과정을 블록 조합으로 표현
- 블록을 보며 "이걸 쓰면 되겠구나" 발견하는 효과
- 디버깅(단계별 실행)을 통한 사고 확장
- Anki 시스템으로 반복 학습

## 설계 결정사항

### 1. 블록 구조

| 구분 | 설명 |
|------|------|
| 공식 블록 | 모든 문제에 범용으로 쓰이는 수학 공식/연산 블록. 주제별 분류. |
| 개별 블록 | 특정 문제의 풀이에서 나오는 중간 계산값(변수). 문제마다 제공. |

- 혼합(C) 방식: 공식 블록 + 연산 블록 + 개별 블록, 계층적으로 분류
- 블록 연결: 수직 스택 + 중첩(블록 안에 블록) + 분기(if 구조)
- 풀이 과정을 충실히 표현하는 데 초점 (단순화하지 않음)
- 사용자 입력: 타이핑 없음, 블록 끼워넣기만

### 2. 실행 모델

- **일괄 실행**: 모든 블록 배치 후 "실행" → 위에서 아래로 계산, 결과 표시 (기본)
- **디버깅 모드**: "다음 단계" 버튼으로 한 블록씩 실행, 중간값 확인 가능

### 3. 채점 방식

- 채점 기준: 최종 답 비교 (풀이 경로 자유)
- 답 제출 후: 모범 블록 풀이를 옆에 표시 (여러 풀이 가능)
- 모범 풀이는 사전에 제작하여 정적 데이터로 포함

### 4. 난이도 단계

| 단계 | 설명 |
|------|------|
| 1단계 | 일부 블록 사전 배치 (빈칸 채우기 형식) |
| 2단계 | 블록 하이라이트 힌트 (사용할 블록 2~3개 강조) |
| 3단계 | 완전 자유 풀이 (힌트 없음) |

### 5. Anki 연동

- 학습한 문제는 자동으로 Anki 복습 대기열에 진입
- 난이도 단계와 Anki 점수 연동:
  - Easy/Good → current_level += 1 (최대 3)
  - Again/Hard → current_level -= 1 (최소 1)
- 3단계 도달 후 유지, 틀리면 내려감

### 6. 네비게이션 (메인 화면)

두 개의 큰 진입점:
- **학습**: 과목 → 단원 → 문제 목록 (처음 푸는 문제)
- **복습**: Anki 기반 순차 출제 (이미 푼 문제, 과목 무관)

### 7. 플랫폼

- MVP: 웹앱 (빠른 프로토타이핑, 반복 수정 용이)
- 최종 목표: 모바일 앱

### 8. 기술 스택

- **Blockly**: 블록 구조, 드래그 앤 드롭, 연결, 중첩
- **KaTeX**: 블록 내부 수식 렌더링
- **커스텀 블록**: 수학 전용 블록 구조 (Blockly에 없는 것)
- 구조 최대한 단순하게 (AI 의존 개발 고려)

### 9. 데이터 저장

- 현재: 브라우저 로컬 스토리지
- 향후: 클라우드 DB + 회원가입 (비용 발생 시점에 전환)

### 10. 앱 내 AI

- 앱 안에 AI 없음 (런타임 API 호출 없음)
- 모든 콘텐츠는 사전 제작하여 정적 데이터로 업로드

## 콘텐츠 파이프라인

### 제작 과정

1. 편입 수학 문제 + 해설 PDF 제공
2. AI가 분석하여 블록 구성 생성:
   - 필요한 공식 블록 선정
   - 모범 풀이의 블록 배치 순서
   - 개별 블록 (중간 계산값)
   - 빈칸 구조 (어디에 뭘 넣는지)
3. 시각적 프로토타입으로 검수 (동작하는 HTML)
4. 확정 후 본 앱에 코드로 통합

### 워크플로우

- 초기: (B) 시각 프로토타입 + 설명 → 검수 → 코드 변환
- 패턴 확립 후: (A) 바로 코드 생성

## MVP 범위

- 과목: 기초수학
- 구체적 범위: PDF 분석 후 결정
- 성공 기준: 기술 동작 확인 + 본인 사용 테스트 + AI 검수

4. 선형대수학
5. 공학수학 1
